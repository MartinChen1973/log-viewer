# 日志数据分析系统需求说明书

## 1. 文档目的与范围

本文档描述基于多系统、多日志源的缺陷与异常聚合分析系统之业务目标与功能边界，供产品设计与研发实现参照。范围涵盖日志接入、分类统计、报告输出及图形化与钻取交互；不包含具体技术栈选型与部署细则。

## 2. 业务目标

将分散在各业务系统中的日志（警告、错误、致命异常等）统一采集与归类，以可阅读的报告形式呈现问题分布、趋势与明细，支撑运维与开发快速定位与优先级判断。

## 3. 核心概念

| 概念     | 说明                                                                   |
| -------- | ---------------------------------------------------------------------- |
| 系统     | 独立业务或部署单元（如服务名、应用标识），作为顶层组织维度。           |
| 日志源   | 属于某一系统下的一条或多条日志文件/流/索引，可配置采集规则。           |
| 条目分类 | 单条日志按级别或规则映射为告警、错误、致命等类别，支持扩展自定义规则。 |

## 4. 功能需求

### 4.1 多系统与日志管理

- 支持注册与维护多个「系统」，每个系统下挂载一个或多个「日志源」。
- 支持按系统、日志源、时间范围筛选数据。

### 4.2 分类与聚合

- 对日志条目按级别（及可选规则）归入固定或自定义类别，统计各类数量与占比。
- 支持按系统、日志源、类别、时间维度聚合（如日/周趋势）。

### 4.3 报告

- 生成结构化报告：摘要（总览）、按系统/日志源/类别分解、可选时间趋势说明。
- 报告可导出或固定链接分享（具体格式可在详细设计中确定）。

### 4.4 图形化展示

- 提供图表展示，例如：各类别占比（饼图/环形）、时间序列趋势（折线/柱状）、系统间对比（柱状/条形）。
- 图表与筛选条件联动，数据更新一致。

### 4.5 钻取（Drill-down）

- 从汇总视图逐级下钻：例如 系统 → 日志源 → 类别 → 具体日志条目列表。
- 钻取路径可回退；列表支持关键字或规则过滤（可分期实现）。

### 4.6 可选扩展（非首期必选）

- 与工单或缺陷系统关联（外部 ID、跳转链接）。
- 异常模式聚类或简单关键词热点（需单独评估数据源与算法）。

## 5. 非功能需求（概要）

- **可用性**：界面层次清晰，主要报表与钻取路径在少数步骤内可达。
- **性能**：在约定数据量级下，常用汇总与钻取查询响应可接受（具体指标在详细设计中量化）。
- **安全与合规**：日志可能含敏感信息时需脱敏、权限控制与审计要求另文规定。

## 6. 验收要点（摘要）

- 多系统、多日志源模型可用且数据归属正确。
- 分类统计与报告内容一致；图表与表格数据一致。
- 钻取路径完整，末端可定位到具体日志条目（或等价明细）。

## 7. 领域实体（数据模型拆分）

以下为逻辑实体，物理表结构可在详细设计中落地；命名以业务可读性优先。

| 实体                                        | 唯一标识                  | 主要属性（示例）                                                             | 说明                                                         |
| ------------------------------------------- | ------------------------- | ---------------------------------------------------------------------------- | ------------------------------------------------------------ |
| **应用系统** `ApplicationSystem`            | `system_id`               | 名称、编码、描述、排序、启用状态                                             | 顶层组织单元，对应「不同系统」。                             |
| **日志源** `LogSource`                      | `log_source_id`           | 所属 `system_id`、名称、类型（文件/流/API）、路径或连接信息、采集状态        | 某系统下的一个可分析日志入口；类型细节见详细设计。           |
| **解析/分类规则** `CategoryRule`（可选）    | `rule_id`                 | 所属范围（全局/系统/日志源）、匹配模式、映射类别、优先级                     | 将原始行映射到「条目分类」；可无表而以配置固定级别规则起步。 |
| **日志条目** `LogEntry`                     | `entry_id` 或（源内偏移） | `log_source_id`、时间戳、原始级别、归一分类、摘要、存储引用（文件行号/偏移） | 钻取末端对象；大数据量时可仅索引+延迟加载正文。              |
| **条目分类** `IssueCategory`                | `category_code`           | 显示名、排序、严重度权重                                                     | 枚举或字典表，如：告警、错误、致命、信息（是否展示可配置）。 |
| **聚合统计** `AggregationSlice`（计算视图） | —                         | 维度（系统/日志源/分类/日桶等）、计数、占比、时间桶起止                      | 非必落库；可由查询实时计算或物化缓存。                       |
| **分析报告** `AnalysisReport`               | `report_id`               | 时间范围、筛选条件快照、生成时间、状态、导出文件引用（若有）                 | 对应一次「定稿」报告，便于分享与对照。                       |

**实体关系（概要）**：`ApplicationSystem` 1 — N `LogSource`；`LogSource` 1 — N `LogEntry`；`LogEntry` 经规则映射到 `IssueCategory`；`AnalysisReport` 引用查询条件而非必须复制全量条目。

## 8. 前端页面与路由（信息架构）

以下为建议页面与示例路由，实施时可按单页应用（SPA）或传统多页调整；职责与数据依赖一并列出。

| 页面                     | 示例路由                                                      | 主要职责                                                     | 依赖数据（实体/API）          |
| ------------------------ | ------------------------------------------------------------- | ------------------------------------------------------------ | ----------------------------- |
| **总览仪表盘**           | `/` 或 `/dashboard`                                           | 多系统 KPI、全局类别占比图、趋势缩略图；作为钻取起点。       | 汇总聚合、系统列表。          |
| **应用系统列表/维护**    | `/systems`                                                    | 系统增删改查（权限内）、进入某系统分析。                     | `ApplicationSystem`。         |
| **系统分析**             | `/systems/:systemId`                                          | 单系统下日志源列表、该系统内分类统计与小图；下钻到源或类别。 | `LogSource`、按系统聚合。     |
| **日志源浏览**           | `/systems/:systemId/sources/:sourceId`                        | 源信息、时间范围筛选、类别分布、进入条目列表。               | `LogSource`、聚合、条目查询。 |
| **分类明细（钻取）**     | `/systems/:systemId/sources/:sourceId/category/:categoryCode` | 指定类别下的条目列表、分页/搜索、跳转全文。                  | `LogEntry`、过滤条件。        |
| **日志条目全文/上下文**  | `/entries/:entryId` 或抽屉/侧栏                               | 展示单条内容及前后文（若可获取）。                           | `LogEntry` 详情。             |
| **分析报告**             | `/reports`                                                    | 报告列表、生成新报告、打开历史报告视图。                     | `AnalysisReport`。            |
| **报告阅读**             | `/reports/:reportId`                                          | 静态或准静态报告版式：摘要、图表、表格与钻取链接。           | 报告快照 + 聚合/条目 API。    |
| **分类规则配置**（可选） | `/settings/rules`                                             | 维护 `CategoryRule` 与测试匹配用例。                         | `CategoryRule`。              |

图表与筛选控件应复用同一套查询参数（时间范围、`system_id`、`log_source_id`、`category`），以保证列表、图、报告一致。

## 9. HTTP API 一览（REST 风格草案）

基路径记为 `/api`；以下为资源与典型动词。具体查询参数（分页游标、`from`/`to`、排序）在接口说明中补全。首期为只读分析亦可仅实现 `GET`。

### 9.1 配置与目录

| 方法    | 路径                                   | 说明                     |
| ------- | -------------------------------------- | ------------------------ |
| `GET`   | `/api/systems`                         | 应用系统列表。           |
| `POST`  | `/api/systems`                         | 新建系统（需写权限时）。 |
| `GET`   | `/api/systems/{system_id}`             | 系统详情。               |
| `PATCH` | `/api/systems/{system_id}`             | 更新系统元数据。         |
| `GET`   | `/api/systems/{system_id}/log-sources` | 某系统下日志源列表。     |
| `POST`  | `/api/systems/{system_id}/log-sources` | 注册日志源。             |
| `GET`   | `/api/log-sources/{log_source_id}`     | 日志源详情。             |
| `PATCH` | `/api/log-sources/{log_source_id}`     | 更新日志源配置。         |

### 9.2 聚合、图表与钻取

| 方法  | 路径                             | 说明                                                                                                            |
| ----- | -------------------------------- | --------------------------------------------------------------------------------------------------------------- | --------------- | ----------- |
| `GET` | `/api/aggregations/summary`      | 全局或按筛选条件的摘要：总条数、按 `IssueCategory` 计数等。查询参数：`system_id`、`log_source_id`、时间范围等。 |
| `GET` | `/api/aggregations/by-dimension` | 按维度分组（如按系统、按日志源）；用于柱状对比。参数：`dimension=system                                         | log_source      | category`。 |
| `GET` | `/api/aggregations/timeseries`   | 时间序列桶（日/小时）；参数：`bucket=1d                                                                         | 1h`、筛选条件。 |
| `GET` | `/api/log-entries`               | 条目列表（钻取末端）；过滤：`system_id`、`log_source_id`、`category_code`、`q`（关键词）、时间范围、`cursor`。  |
| `GET` | `/api/log-entries/{entry_id}`    | 单条条目详情与可选上下文。                                                                                      |

### 9.3 报告

| 方法   | 路径                              | 说明                                                     |
| ------ | --------------------------------- | -------------------------------------------------------- |
| `GET`  | `/api/reports`                    | 历史报告列表。                                           |
| `POST` | `/api/reports`                    | 按当前筛选条件生成报告（请求体携带时间范围与维度选项）。 |
| `GET`  | `/api/reports/{report_id}`        | 报告元数据与内容快照（或结构化块）。                     |
| `GET`  | `/api/reports/{report_id}/export` | 导出 PDF/HTML（若实现）。                                |

### 9.4 规则（可选）

| 方法     | 路径                            | 说明                                                |
| -------- | ------------------------------- | --------------------------------------------------- |
| `GET`    | `/api/category-rules`           | 规则列表；可按 `system_id` / `log_source_id` 过滤。 |
| `POST`   | `/api/category-rules`           | 新建规则。                                          |
| `PATCH`  | `/api/category-rules/{rule_id}` | 更新规则。                                          |
| `DELETE` | `/api/category-rules/{rule_id}` | 删除规则。                                          |

**说明**：与现有示例工程中带前缀的日志文件 API 可并行存在；新资源建议统一挂在 `/api` 下并逐步收敛命名。跨域、认证与速率限制不在本文展开。
